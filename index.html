The OSI (Open Systems Interconnection) Model is a conceptual framework used to understand and implement interoperable network protocols. It is divided into 7 layers, each with specific functions.

OSI Model Layers - Interactive Diagram

Application Layer

Provides network services to applications. Example: HTTP, FTP.

This layer interacts with software applications and provides network services to end-user processes.

Presentation Layer

Translates data formats, encryption. Example: SSL/TLS.

It formats or translates data for the application layer, including encryption and compression.

Session Layer

Manages sessions, connections. Example: NetBIOS.

Establishes, manages, and terminates sessions between communicating hosts.

Transport Layer

Reliable data transfer, error correction. Example: TCP, UDP.

Ensures error-free delivery of data with flow control and segmentation.

Network Layer

Routing, IP addressing. Example: IP.

Responsible for logical addressing and path determination (IP addressing).

Data Link Layer

MAC addressing, error detection. Example: Ethernet.

Provides node-to-node data transfer and error checking (MAC addresses).

Physical Layer

Physical transmission of data. Example: Cables, Hubs.

Defines electrical, mechanical, and procedural aspects of physical communication.

The TCP/IP Model

The TCP/IP model is another framework used to understand network communication. It is a simpler model than the OSI model, consisting of four layers:

  • **Application Layer:** (Corresponds to OSI Application, Presentation, and Session Layers) - Provides network services to applications. Protocols include HTTP, FTP, SMTP, DNS.
  • **Transport Layer:** (Corresponds to OSI Transport Layer) - Provides end-to-end communication services. Protocols include TCP and UDP.
  • **Internet Layer:** (Corresponds to OSI Network Layer) - Handles logical addressing and routing of packets. The primary protocol is IP.
  • **Network Access Layer:** (Corresponds to OSI Data Link and Physical Layers) - Deals with the physical transmission of data and local network addressing (MAC addresses). Examples include Ethernet and Wi-Fi.

While the OSI model is more theoretical and detailed, the TCP/IP model is widely used in practice due to its direct relationship with the internet protocols.
